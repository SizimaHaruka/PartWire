# PartWire 基本設計書

## 1. 目的

本書は、要件定義書をもとに `PartWire` の基本設計方針を整理するための初版である。現時点では、少人数運用に適したシンプルな構成を優先し、後から拡張できる設計を目指す。

## 2. 設計方針

- 案件を中心に、見積、承認、発注、納品、請求、回付を時系列で管理する
- 工事番号と案件番号を分離し、工事番号は最上位集計単位、案件はその配下の業務運用単位として扱う
- 部品マスタと価格履歴を分離し、価格の変遷を蓄積する
- 購入部品と製作部品を同一業務フローで扱えるようにしつつ、属性は別管理とする
- 納品確認を最重要操作とし、一覧性と入力効率を優先する
- CSV 取込を標準機能とし、マスタ整備と既存データ移行に対応する
- 将来的な添付ファイル管理や通知機能の追加を想定し、拡張余地を持たせる
- 見積書は取引単位として扱い、進捗と完了判定は明細レベルで積み上げる

## 3. 想定アプリ構成

現行リポジトリの構成から、まずはデスクトップアプリとして開始する前提で整理する。

- クライアント: WPF
- 永続化: SQLite または SQL Server を想定
- 入出力: 画面操作、CSV 取込、CSV 出力
- 配布: インストーラー

補足:
- 実運用人数が少ないため、初期段階は過度に複雑な分散構成を避ける
- 確認用のローカルスタンドアロン版は SQLite を想定する
- 複数人同時利用を行う場合は SQL Server 系共有 DB を想定する
- 認証方式はローカル認証と AD 認証の切り替えを想定する
- Active Directory 認証では Windows ログオン連携を想定する

### 5.5 承認ルート決定方針

- 案件または見積書に設定された部署をもとに承認ルート候補を抽出する
- 見積金額から金額ランクを判定する
- 部署と金額ランクに対応する承認ルート定義から必要承認段階を決定する
- 承認者候補は独自ユーザー情報または AD グループ参照結果から解決する
- 必要に応じて案件個別に承認者差し替えを許容する
- 見積書が改版された場合は、旧版の承認結果を引き継がず再承認を必要とする

### 5.6 見積書改版時の扱い

- 見積書改版時は旧版見積書をキャンセル扱いとする
- 旧版に紐づく未発注データは新しい版へ引き継がない
- 旧版に紐づく発注済データがある場合は、発注キャンセルを記録して履歴を保持する
- 新版見積書は別途承認を取り直したうえで発注する

### 5.7 認証方針

- システム設定で AD 認証またはローカル認証を切り替える
- 認証方式に関わらず、業務上の利用者レコードを共通で利用する
- AD 認証時はログインIDまたは AD 連携キーを用いて利用者を特定する
- ローカル認証時はログインIDとパスワードハッシュで認証する
- AD グループ参照は認証方式とは独立して有効化可否を設定できるようにする

目的:
- 最小操作で納品確認を完了する

設計方針:

- サプライヤー、納品日、案件番号、納品書番号で素早く絞り込めるようにする
- 一覧上で確認数量、差異、確認結果を直接入力できるようにする
- キーボード主体で登録できるようにする
- 部分納品や差異ありをその場で記録できるようにする
- 見積書単位の進捗と、明細ごとの残数量を同時に把握できるようにする
- 納品書単位で対象を開き、1 件ずつ確認完了できる導線を優先する

### 6.5 納品書登録画面

目的:
- 見積書をもとに納品書を素早く登録する

設計方針:
- 対象見積書を選択して、その明細を初期表示する
- 見積書明細ごとの残納数量を表示する
- 納品数量を明細ごとに入力できるようにする
- 分納済み実績を参照しながら入力できるようにする
- 納品書番号、納品日、添付リンクをあわせて登録できるようにする
- 購入部品ではサプライヤー候補を、製作部品では加工メーカー候補を優先表示する

### 6.6 請求書登録画面

目的:
- 納品実績をもとに請求書を素早く登録する

設計方針:
- 対象納品書または未請求納品明細を選択して初期表示する
- 納品明細ごとの未請求数量、未請求金額を表示する
- 今回請求対象とする数量、金額を入力できるようにする
- 月またぎ分納や分割請求の履歴を参照しながら入力できるようにする
- 請求書番号、請求日、支払対象月、添付リンクをあわせて登録できるようにする

## 11. 技術設計上の追加方針

- DB 製品差異はデータアクセス層で吸収する
- SQLite から SQL Server への移行手段を用意する
- 主要テーブルには楽観ロック用の更新トークンまたは更新日時を持たせる
- 添付保存先は設定変更可能とし、標準は共有フォルダとする
- サンプル読み込み機能により初期データ投入を行えるようにする
